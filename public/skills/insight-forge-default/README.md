# Insight Forge — Skills 번들: `insight-forge-default`

본 번들은 Skills.md 기반 범용 투자 분석 대시보드 시스템 **Insight Forge**의 기본 분석 규칙 모음이다. 5개의 마크다운 문서가 하나의 묶음으로 작동하여, 투자 데이터로부터 자동으로 정규화·계산·시각화·진단·리포트 구성까지 수행한다.

**핵심 컨셉**: Skills.md는 사용자에게는 "분석 규칙 문서"이고, AI 에이전트에게는 "프롬프트"다. 동일 문서가 두 가지 독자(사람과 기계) 모두에게 의미를 갖도록 설계되었다.

---

## 1. 번들 구성

본 번들은 5개의 Skills.md 파일로 구성되며, 대회가 정의한 5가지 Skills.md 역할을 모두 충족한다.

| 파일 | 대회 정의 역할 | 의존성 |
|------|--------------|--------|
| `data-rules.md` | 투자 데이터 **분석 기준** 정의 | (없음) |
| `metric-rules.md` | 투자 **지표 계산 규칙** 정의 | `data-rules` |
| `viz-rules.md` | **시각화 선택 기준** 정의 | `data-rules`, `metric-rules` |
| `insight-rules.md` | 투자 **인사이트 생성 규칙** 정의 | `data-rules`, `metric-rules` |
| `report-rules.md` | **리포트 구성 흐름** 설계 | 모든 파일 |
| `README.md` | (본 문서) 번들 전체 개요 및 사용법 | - |

---

## 2. 처리 파이프라인

번들의 5개 파일은 다음 6단계 파이프라인에서 사용된다.

```
[Raw 투자 데이터]
       ↓
   ① 데이터 입력
       ↓
   ② 정규화                ← data-rules.md
       ↓
   ③ 지표 계산              ← metric-rules.md
       ↓
   ④ 시각화 선택            ← viz-rules.md
       ↓
   ⑤ 인사이트 생성          ← insight-rules.md
       ↓
   ⑥ 리포트 구성·렌더링      ← report-rules.md
       ↓
[최종 대시보드 / PDF 리포트]
```

각 단계는 해당 Skills.md 파일의 규칙에 따라 동작한다. 코어 시스템은 이 파일들을 교체·수정하는 것만으로 분석 결과를 변경할 수 있다.

## 3. 본 번들의 차별점

### 3.1 Insight Forge 자체 정의 지표 5종

`metric-rules.md`에는 본 시스템이 직접 정의한 5가지 차별화 지표가 포함되어 있다.

1. **분산투자 점수** (Diversification Score, 0~100)
2. **집중도 점수** (Concentration Score, HHI 기반 - 미국 법무부 반독점 기준 응용)
3. **포트폴리오 건강도** (Health Score, 4개 핵심 지표 가중 평균)
4. **위험조정수익률** (RAR, 단순화된 샤프지수)
5. **모멘텀 강도** (Momentum Strength, 추세 가속도)

이 지표들은 기존 도구에서 제공하지 않거나, 제공하더라도 직관적으로 해석하기 어려운 형태로만 노출되는 경우가 많다. 본 번들은 이를 0~100점 또는 명확한 범위로 정규화하여 일반 투자자도 즉시 이해할 수 있게 한다.

### 3.2 자연어 인사이트 자동 생성

`insight-rules.md`는 22개의 트리거 조건을 정의하며, 각 조건이 충족될 때 자동으로 자연어 메시지를 생성한다. 단순히 "변동성이 32%입니다"가 아니라 "변동성이 벤치마크 대비 1.6배 높으니 비중 조정을 검토하세요"와 같이 **다음 행동을 제안**하는 형태다.

### 3.3 시나리오 기반 리포트 구성

`report-rules.md`는 동일 데이터에 대해 **사용자 시나리오에 따라 다른 대시보드**를 자동 구성한다. 5개 시나리오(개인 기본·리스크 중심·성과 중심·컴플라이언스·임원 브리핑)를 지원한다.

### 3.4 LLM 친화적 구조

모든 규칙은 마크다운 헤딩, 테이블, 코드블록, YAML frontmatter로 구조화되어 있다. 이는 다음을 가능케 한다.

- LLM(Cursor, Claude 등)이 본 번들을 컨텍스트로 받아 React 컴포넌트 코드를 자동 생성
- 별도 임베딩 없이 자연어 질의를 본 규칙에 대해 직접 수행
- 사람이 읽기에도 가독성 우수

## 4. 적용 대상 데이터

본 번들은 다음 형태의 투자 데이터에 적용 가능하다.

| 데이터 유형 | 예시 | 처리 가능 여부 |
|------------|------|---------------|
| 국내 주식 시계열 | 삼성전자, NAVER, 카카오 일별 시세 | ✅ |
| 국내 ETF 시세 | KODEX 200, TIGER 미국S&P500 | ✅ |
| 시장 지수 | KOSPI, KOSDAQ, S&P500 | ✅ (벤치마크) |
| 사용자 포트폴리오 | 보유 종목·비중 정보 | ✅ |
| 해외 주식 (USD) | 환율 변환 후 적용 | [주의] 가능하나 전용 규칙 권장 |
| 채권 | 만기·쿠폰 등 별도 필드 필요 | [불가] 본 번들 범위 외 |
| 암호화폐 | 24시간 거래·온체인 지표 | [불가] 별도 번들(`insight-forge-crypto`) 사용 |

## 5. 활용 시나리오

본 번들은 개인 투자자와 금융 기업 양쪽에서 활용 가능하다.

### 5.1 개인 투자자 활용

**시나리오**: 30대 직장인이 본인 포트폴리오를 점검

1. CSV/JSON으로 보유 종목 데이터 업로드
2. 시나리오 = `personal_default` 자동 적용
3. 종합 진단 → KPI → 추세 → 구성 → 인사이트 순으로 대시보드 표시
4. "분산투자 점수가 35점이니 다른 섹터 종목 추가를 검토하세요" 같은 액션 제안 수신

### 5.2 금융 기업 활용 — 자산운용팀

**시나리오**: 자산운용사의 포트폴리오 매니저가 운용 중인 펀드를 모니터링

1. 운용 중인 펀드의 보유종목 데이터 입력
2. 시나리오 = `performance_focused` 적용
3. 알파·정보비율·추적오차 등 운용 성과 지표 강조
4. `insight-rules.md`의 임계값을 사내 기준으로 커스터마이징 가능
   - 예: 변동성 경고 임계값 30% → 사내 가이드라인에 따라 25%로 조정
5. 매일 자동 생성되는 PDF 리포트를 운용회의 자료로 사용

### 5.3 금융 기업 활용 — 리스크 관리팀

**시나리오**: 금융기관의 리스크 매니저가 클라이언트 포트폴리오를 점검

1. 다수 클라이언트의 포트폴리오를 일괄 분석
2. 시나리오 = `risk_focused` 적용
3. 변동성·MDD·VaR·집중도 강조 표시
4. 단일 종목 비중 30% 초과 등 컴플라이언스 임계 자동 감지
5. 위험 인사이트만 필터링하여 우선순위별로 정렬

### 5.4 금융 기업 활용 — 컴플라이언스 검토

**시나리오**: 컴플라이언스 담당자가 포트폴리오 규제 준수 여부 점검

1. 시나리오 = `compliance_review` 적용
2. 단일종목 비중·섹터 노출도·집중도 자동 검사
3. 규제 임계값 초과 시 critical 인사이트 즉시 트리거
4. PDF 리포트로 출력하여 감사 자료로 보관 (적용된 Skills.md 원문 첨부됨)

### 5.5 금융 기업 활용 — 임원 브리핑

**시나리오**: C-레벨 임원이 1분 안에 핵심만 파악

1. 시나리오 = `executive_brief` 적용
2. 종합 진단 영역만 표시, 나머지 모두 숨김
3. 포트폴리오 건강도 1개 게이지 + 핵심 메시지 1줄
4. 모바일에서도 한 화면 내 즉시 확인

## 6. 번들 교체로 도메인 확장

본 시스템의 가장 큰 강점은 **번들 교체만으로 다른 분석 도메인으로 전환** 가능하다는 점이다.

| 번들 | 적용 시 결과 | 코어 수정 | 제공 여부 |
|-----|--------------|----------|----------|
| `insight-forge-default` | 개인 투자자·금융 기업용 | - | 포함 |
| `insight-forge-crypto` | 암호화폐 변동성·온체인 분석 | 불필요 | 포함 |
| `insight-forge-esg` | ESG 점수 기반 종목 스크리닝 | 불필요 | 향후 |
| `insight-forge-fund` | 펀드 운용 성과 평가 | 불필요 | 향후 |
| `insight-forge-bond` | 채권 듀레이션·신용 분석 | 불필요 | 향후 |

**동일한 코어 시스템, 다른 규칙.** 이것이 Insight Forge가 단순한 "투자 대시보드 도구"가 아닌 "투자 대시보드 플랫폼"인 이유다.

제출물에는 `insight-forge-default`(주력)와 `insight-forge-crypto`(확장 시연용) 두 번들이 함께 들어 있어, 재사용 가능성을 실물로 증명한다.

## 7. 사용 예시

### 7.1 본 번들의 적용 흐름

1. 사용자가 보유 포트폴리오 데이터(CSV/JSON)를 업로드
2. `data-rules.md`에 따라 자동 정규화 (컬럼 매핑, 결측치 처리, 날짜 통일, 데이터 품질 점수 산출)
3. `metric-rules.md`에 따라 13개 이상의 지표 자동 계산 (자체 지표 5종 포함)
4. `viz-rules.md`에 따라 각 지표가 적합한 차트 컴포넌트로 매핑
5. `insight-rules.md`의 22개 조건을 평가하여 트리거된 인사이트 카드 생성
6. `report-rules.md`에 따라 시나리오별 위젯 배치 및 내러티브 흐름 구성
7. 최종 대시보드 렌더링 (KPI · 차트 · 인사이트 카드 통합)

### 7.2 출력 미리보기

```
[O] 포트폴리오 건강도: 67점 (양호)
═══════════════════════════════════

[수익률 +14.2%] [변동성 21.8%] [샤프 1.12] [MDD -15.6%]

[추세] 누적 수익률 추이              [비중] 자산 비중
   (KOSPI 대비 +3.4%p)            삼성전자 40%
                                 NAVER    30%
                                 카카오   30%

[히트맵] 상관계수 히트맵              [차트] 종목별 수익률
   (3×3 매트릭스)                삼성전자 +18%
                                NAVER    +11%
                                카카오   +14%

Tip: 실행 가능 인사이트
   [O] 샤프지수 1.12 — 위험 대비 수익률 양호
   [!] 분산투자 점수 58점 — 보통 수준
   [i] 모멘텀 강도 +3.2pt — 약한 상승 추세
```

## 8. 라이선스 및 기여

Insight Forge의 Skills 번들은 해커톤 출품작으로 제출된 작업물이다. 본 번들의 구조를 참고하여 자체적인 분석 규칙 번들을 작성하는 것을 환영한다.

### 8.1 새 번들 작성 시 권장 사항

- 본 번들의 5파일 구조(`data` / `metric` / `viz` / `insight` / `report`)를 그대로 따를 것
- 각 파일은 frontmatter로 메타데이터 명시
- 의존성(`depends_on`)을 정확히 기재하여 검증 자동화 가능하게 함
- 자체 지표를 정의할 때는 **명확한 수식 + 해석 기준 + 차별성 설명**을 함께 기재
- 사용자 시나리오를 미리 정의하여 도메인 특성 반영

### 8.2 버전 관리

| 버전 | 일자 | 변경 사항 |
|------|------|-----------|
| 1.0.0 | 2026-04-29 | 최초 작성 — 5개 파일, 자체 지표 5종, 인사이트 규칙 22종, 시나리오 5종 포함 |
