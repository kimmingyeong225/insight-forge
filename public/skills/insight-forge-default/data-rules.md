# 데이터 정규화 규칙 (data-rules)

이 문서는 다양한 형식의 raw 투자 데이터를 Insight Forge 시스템이 인식하는 **표준 스키마**로 변환하는 규칙을 정의한다. 코어 시스템은 이 규칙을 읽어 자동으로 데이터를 매핑·검증한다.

---

## 1. 표준 스키마 정의

### 1.1 시계열 데이터 (Time Series)

종목·지수·ETF의 일별 가격 데이터에 적용한다.

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| `date` | string (YYYY-MM-DD) | ✅ | 거래일자 |
| `symbol` | string | ✅ | 종목 식별자 (티커 또는 종목코드) |
| `close` | number | ✅ | 종가 |
| `open` | number | [불가] | 시가 |
| `high` | number | [불가] | 고가 |
| `low` | number | [불가] | 저가 |
| `volume` | integer | [불가] | 거래량 |

### 1.2 포트폴리오 데이터 (Portfolio)

사용자의 보유 자산 정보에 적용한다.

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| `symbol` | string | ✅ | 종목 식별자 |
| `weight` | number (0~1) | ✅ | 포트폴리오 내 비중 (소수) |
| `quantity` | integer | [불가] | 보유 수량 |
| `avg_price` | number | [불가] | 평균 매수단가 |

**제약 조건**: 포트폴리오 내 모든 `weight`의 합은 1.0 ± 0.01 이어야 한다.

**예외 처리 및 복원 동작**:
- 합계가 1.0 ± 0.01 범위를 벗어나면 사용자에게 **경고 메시지를 표시**한다
- 동시에 모든 `weight` 값을 합계로 나누어 **총합이 1.0이 되도록 강제 스케일링(정규화)하여 렌더링**한다
- 정규화 공식: `normalized_weight[i] = weight[i] / Σ(weight)`
- 이를 통해 도넛 차트 등 비중 기반 시각화가 깨지지 않고 정상 렌더링된다
- 정규화된 값은 사용자에게 표시하되, 원본 입력값도 함께 보존하여 사용자가 수정할 수 있게 한다

### 1.3 시장 지수 데이터 (Market Index)

벤치마크 비교용 지수 데이터에 적용한다. 시계열 스키마와 동일하나 `symbol`은 지수명을 사용한다 (예: `KOSPI`, `KOSDAQ`, `S&P500`).

## 2. 컬럼 매핑 규칙

Raw 데이터의 다양한 컬럼명을 표준 필드명으로 매핑한다. 매핑은 **대소문자 무시**, **공백 제거** 후 일치 여부를 판단한다.

### 2.1 시계열 컬럼 매핑

```yaml
date:
  - "date"
  - "날짜"
  - "일자"
  - "기준일"
  - "trade_date"
  - "거래일"

symbol:
  - "symbol"
  - "ticker"
  - "종목코드"
  - "종목명"
  - "code"

close:
  - "close"
  - "종가"
  - "closing_price"
  - "끝값"
  - "현재가"

open:
  - "open"
  - "시가"
  - "opening_price"
  - "시작값"

high:
  - "high"
  - "고가"
  - "최고가"

low:
  - "low"
  - "저가"
  - "최저가"

volume:
  - "volume"
  - "거래량"
  - "vol"
  - "trade_volume"
```

### 2.2 매핑 우선순위

동일 컬럼이 여러 후보와 매치될 경우 다음 우선순위를 따른다.

1. **완전 일치** > 부분 일치
2. **영문 키워드** > 한글 키워드 (국제 표준 우선)
3. **목록 상단** > 하단 (위에 있는 것이 우선)

## 3. 날짜 형식 정규화

### 3.1 지원 입력 형식

다음 형식들을 모두 자동 인식하여 `YYYY-MM-DD` 표준 형식으로 변환한다.

| 입력 형식 | 예시 | 변환 결과 |
|-----------|------|-----------|
| `YYYY-MM-DD` | `2026-04-29` | `2026-04-29` |
| `YYYY/MM/DD` | `2026/04/29` | `2026-04-29` |
| `YYYYMMDD` | `20260429` | `2026-04-29` |
| `YYYY.MM.DD` | `2026.04.29` | `2026-04-29` |
| `YYYY년 MM월 DD일` | `2026년 4월 29일` | `2026-04-29` |
| `MM/DD/YYYY` | `04/29/2026` | `2026-04-29` |

### 3.2 변환 실패 시 처리

- 인식 불가능한 형식 발견 시 해당 행을 **로그에 기록하고 제외**한다.
- 전체 행의 5% 이상이 변환 실패하면 **에러를 발생**시키고 사용자에게 알림.

## 4. 결측치 처리 규칙

### 4.1 필드별 처리 방식

| 필드 | 결측치 처리 방식 | 비고 |
|------|------------------|------|
| `close` | **직전 영업일 값으로 채움** (forward fill) | 가격은 연속성이 중요 |
| `open`, `high`, `low` | 동일 행의 `close` 값으로 대체 | 단일 거래일 추정 |
| `volume` | **0으로 채움** | 거래 없음으로 간주 |
| `date` | **해당 행 제거** | 식별자 없이는 의미 없음 |
| `symbol` | **해당 행 제거** | 식별자 없이는 의미 없음 |
| `weight` | **에러 발생** | 포트폴리오 무결성 위반 |

### 4.2 처리 전략 선택

`data-rules.md`의 다음 설정으로 전체 전략을 변경할 수 있다.

```yaml
missing_value_strategy:
  numeric_default: forward_fill   # forward_fill | linear_interpolation | drop_row | zero_fill
  identifier_missing: drop_row     # drop_row | use_unknown
  threshold_drop: 0.05             # 결측 비율이 이 값을 초과하면 컬럼 자체 제거
```

## 5. 이상치 검증 규칙

### 5.1 가격 이상치

- `close <= 0`: **에러 발생**
- 전일 대비 변동률 > 30% 또는 < -30%: **경고만 발생** (상한가/하한가 가능성)
- `high < low` 또는 `high < close`: **에러 발생** (논리 오류)

### 5.2 거래량 이상치

- `volume < 0`: **에러 발생**
- `volume == 0`이 30일 이상 지속: **경고 발생** (거래정지 의심)

---

## 6. 데이터 품질 점수

정규화 후 데이터 품질을 0~100점으로 평가하여 사용자에게 표시한다.

```
quality_score = 100 
              - (결측치_비율 × 50)
              - (이상치_비율 × 30)
              - (날짜_불연속_비율 × 20)
```

- 90점 이상: ✅ 정상
- 70~89점: [주의] 주의 (분석 가능하나 일부 신뢰도 저하)
- 70점 미만: [불가] 사용 권장하지 않음

## 7. 적용 예시

### 7.1 Raw 입력 (CSV)

```
날짜,종목코드,종가,거래량
2026/04/01,005930,72500,15234567
2026/04/02,005930,72800,
2026/04/03,005930,,12345678
```

### 7.2 정규화 결과 (JSON)

```json
[
  {"date": "2026-04-01", "symbol": "005930", "close": 72500, "volume": 15234567},
  {"date": "2026-04-02", "symbol": "005930", "close": 72800, "volume": 0},
  {"date": "2026-04-03", "symbol": "005930", "close": 72800, "volume": 12345678}
]
```

처리 내역:
- 날짜 형식 `YYYY/MM/DD` → `YYYY-MM-DD` 변환
- 컬럼명 한글 → 표준 영문명 매핑
- 2번째 행 `volume` 결측 → 0으로 채움
- 3번째 행 `close` 결측 → 직전값(72800) 사용
