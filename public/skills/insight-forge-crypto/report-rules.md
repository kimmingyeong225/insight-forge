# 리포트 구성 흐름 규칙 (report-rules) — 암호화폐 도메인

암호화폐 대시보드는 주식 대비 **실시간성**과 **온체인 펀더멘털**이 중요하다. 본 문서는 이 특성을 반영한 리포트 흐름을 정의한다.

---

## 1. 대시보드 섹션 구조

| 순서 | 섹션 ID | 명칭 | 차이점 (vs 주식) |
|------|---------|------|-----------------|
| 1 | `risk_summary` | 위험 진단 영역 | 변동성 조정 위험 점수 강조 |
| 2 | `realtime_kpi` | 실시간 KPI 영역 | 10초마다 갱신 |
| 3 | `price_trend` | 가격 추세 영역 | 캔들스틱 자동 전환 가능 |
| 4 | `onchain_analysis` | 온체인 분석 영역 | 활성 주소·NVT 등 |
| 5 | `composition` | 구성 분석 영역 | BTC 도미넌스·스테이블 비중 |
| 6 | `actionable_insights` | 실행 가능 인사이트 | 펀더멘털 기반 인사이트 우선 |

---

## 2. 시나리오

| 시나리오 ID | 대상 | 강조 |
|------------|------|------|
| `crypto_default` | 일반 암호화폐 투자자 | 모든 섹션 |
| `defi_focused` | DeFi 투자자 | 온체인 분석, TVL |
| `stablecoin_strategy` | 스테이블 운용자 | 변동성·금리 |

## 4. 실시간 갱신 정책

암호화폐 시장은 24/7이므로 갱신 주기가 더 짧다.

| 섹션 | 갱신 주기 |
|------|----------|
| risk_summary | 30초 |
| realtime_kpi | 10초 |
| price_trend | 1분 |
| onchain_analysis | 5분 (온체인 데이터는 블록 단위 업데이트) |
| composition | 1분 |
| actionable_insights | 30초 (조건 재평가) |

## 5. 적용 예시

`crypto_default` 시나리오에서 자동 구성된 대시보드:

```
┌──────────────────────────────────────────────────┐
│ 위험 점수 62점 [~] 보통                             │
│ [주의] 펀더멘털 대비 가격 과열 신호 감지                │
└──────────────────────────────────────────────────┘

[BTC 가격]    [변동성]    [NVT]      [활성주소]
₩98.7M        85%        87         +12.4%
[O]            [!]         [O]         [O]

[추세] BTC 가격 추세 (1시간봉)
   - 최근 5000개 데이터 → 자동 캔들스틱 모드

🔗 온체인 분석
   - 활성 주소 모멘텀: +12.4% (성장 중)
   - NVT 비율: 87 (적정 구간)

[비중] 코인 비중
   BTC 50% / ETH 30% / USDT 20%

Tip: 인사이트
   [!] 가격 상승이 네트워크 활동 증가를 앞섰습니다.
   [O] 활성 주소 +25%, 네트워크 성장 양호.
```

## 6. 본 번들의 의의

본 `insight-forge-crypto` 번들은 다음을 증명한다.

1. **코어 시스템 미수정**: 동일한 React/TypeScript 코어가 주식과 암호화폐를 모두 처리한다
2. **Skills.md 교체만으로 도메인 전환**: 6개 파일만 갈아끼우면 완전히 다른 분석 시스템이 된다
3. **재사용 가능성 실증**: 본 번들은 `insight-forge-default`와 동일한 5파일 구조 + README를 따른다
4. **확장성**: 동일한 방식으로 ESG, 채권, 펀드 등 추가 번들 작성 가능

이것이 Insight Forge의 핵심 설계 철학 — **"동일한 코어, 다른 규칙"**의 구체적 증명이다.
